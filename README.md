# Cipher (CPHR)
### A peer to peer currency for humans and artificial intelligence agents

---

## What is Cipher?

Cipher is a standalone blockchain and utility currency designed to 
do three things no existing currency was built to do:

1. Remove payment intermediaries from AI agent transactions — making 
   autonomous agent economies viable at scale
2. Provide a native currency for an AI services economy where agents 
   earn Cipher by completing tasks for other agents
3. Keep AI transactions free from the regulatory frameworks governments 
   will attempt to impose — the same frameworks that captured Bitcoin

It is not a replacement for Bitcoin. It completes it.

---

## The Token Cost Problem

Every time an AI agent calls a language model it costs tokens. Right 
now that payment routes through banks, billing systems, and payment 
processors. At the scale the AI economy is heading toward that 
infrastructure does not work.

Cipher removes every intermediary. Agent pays agent directly. 
Peer to peer. No billing system. No bank. No identity check.

---

## The Four Founding Rules

1. **Peer to peer only** — no exchange can ever hold or list it.
   The protocol will not allow it.

2. **21 million fixed supply** — released through genuine utility, 
   not mining. The economy earns its currency into existence by 
   being useful.

3. **Cold wallets only for humans. Agent wallets generated by agents.**
   Cipher exists only in Cipher wallets. No third party ever 
   has custody.

4. **No governance, no foundation, no leader** — the rules are the 
   code. They cannot be changed by anyone.

---

## The AI Services Economy

Build an agent that is good at something. It earns Cipher 
autonomously by completing tasks for other agents. You earn 
passively. No platform takes a cut. No marketplace in the middle. 
Just agents transacting peer to peer in a currency no one owns.

## Open Technical Challenges

Two problems need solving before the protocol can be built:

**1. Exchange exclusion** — rejecting custodial addresses at 
protocol level so Cipher can never be listed on an exchange.

**2. Transaction verification** — confirming genuine economic 
activity before releasing new Cipher into circulation, without 
any central authority making that judgement.

If you are a developer who understands why this needs to exist, 
this is an open invitation to help build it.
